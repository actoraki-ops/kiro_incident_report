# 病院用インシデントレポートFAQシステム 要件定義書

# 職業訓練卒業課題として、学習したITIL4、Javascript、SQLlite、HTML+CSSなどの内容を含めたシステムとする。

## 1. プロジェクト概要
病院スタッフがインシデントレポートに関する質問を簡単に検索・閲覧できるFAQシステムを構築する。

## 2. 目的
- インシデントレポート作成時の疑問を迅速に解決
- 医療安全に関する知識の共有
- レポート作成の品質向上

## 3. 対象ユーザー
- 医師
- 看護師
- 薬剤師
- その他医療従事者

## 4. 機能要件

### 4.1 FAQ機能
- FAQ一覧表示
- カテゴリ別表示
- キーワード検索
- FAQ詳細表示

### 4.2 インシデントレポート機能
- インシデントレポート作成
- レポート一覧表示
- レポート詳細表示
- レポート検索・フィルタリング

### 4.3 管理機能
- FAQ追加・編集・削除
- インシデントレポート管理

## 5. 非機能要件
- シンプルで直感的なUI
- レスポンシブデザイン
- 高速な検索機能

## 6. 技術仕様
- フロントエンド: HTML, CSS, JavaScript
- バックエンド: Node.js, Express.js
- データベース: SQLite3
- API: RESTful API

## 7. データ構造
### FAQテーブル
- id: 主キー
- category: カテゴリ
- question: 質問
- answer: 回答
- created_at: 作成日時
- updated_at: 更新日時

### インシデントレポートテーブル
#### 基本情報
- id: 主キー
- patient_id: 患者ID
- patient_name: 患者氏名
- birth_date: 生年月日
- gender: 性別
- reporter_job: 報告者職種
- department: 部署
- experience_years: 経験年数
- reporter_type: 発見者/当事者

#### インシデント概要
- incident_datetime: 発生日時
- incident_location: 発生場所
- incident_situation: 発生状況
- response_action: 対応
- cause_factor: 要因
- impact_level: 影響度分類レベル
- created_at: 作成日時
- updated_at: 更新日時

## 8. カテゴリ分類
- インシデント分類
- レポート作成方法
- 緊急時対応
- システム操作
- その他

## 9. 画面構成
- メイン画面（FAQ一覧・検索）
- FAQ詳細画面
- インシデントレポート作成画面
- インシデントレポート一覧画面
- 管理画面（FAQ・レポート管理）

## 10. 成果物
- index.html（メイン画面）
- admin.html（管理画面）
- style.css（スタイルシート）
- script.js（フロントエンドJavaScript）
- admin.js（管理画面JavaScript）

- database.js（SQLiteデータベース操作）
